# PharmaGuard AI

ML-powered pharmaceutical authenticity verification and supply chain anomaly detection — FastAPI backend + React/Vite/Tailwind frontend.

PharmaGuard AI runs supply chain metadata and distribution transaction records through trained classifiers to flag counterfeit drug batches and anomalous transactions, with full feature-level transparency on every verdict.

## Models

| Model | Task | Endpoint | Output |
|---|---|---|---|
| **Random Forest** | Classify a drug batch as Genuine or Counterfeit using 13 supply chain fields + 5 engineered features (trust score, cold-chain flag, price suspicion, supplier risk) | `POST /api/predict/batch` | `prediction`, `label`, `confidence`, `risk_level`, `engineered_features` |
| **Isolation Forest** | Detect anomalous distribution transactions from 8 fields + 3 engineered ratios (delivery efficiency, cost per unit, risk score) | `POST /api/predict/transaction` | `prediction`, `label`, `anomaly_score` (0–1), `risk_level`, `engineered_features` |
| **CNN** *(optional)* | Authenticate packaging from an uploaded image (64×64 RGB) | `POST /api/predict/image` | `prediction`, `label`, `confidence`, `risk_level` |

Both Random Forest and Isolation Forest are trained on synthetic data generated by `data/generate_dataset.py`. The CNN requires TensorFlow and is skipped automatically if it isn't installed.

## Quick Start

### 1. Backend

```bash
cd backend
pip install -r requirements.txt
python train.py          # generates trained_models/*.pkl (takes <1 min)
uvicorn api.app:app --reload
```

- API: `http://localhost:8000`
- Interactive docs: `http://localhost:8000/docs`
- Health check: `http://localhost:8000/api/health`

> `train.py` must be run once before starting the server — it produces all 6 artifacts in `trained_models/` (2 models + 2 scalers + 2 imputers). The CNN is optional; install `tensorflow` and re-run `train.py` to enable `/api/predict/image`.

### 2. Frontend

```bash
cd frontend
npm install
npm run dev
```

- App: `http://localhost:5173`
- `vite.config.js` proxies `/api/*` to `http://localhost:8000`, so the backend must be running.

## API Reference

### `POST /api/predict/batch`
```json
{
  "supplier_id": 5,
  "temperature_log": 4.8,
  "humidity_log": 44.5,
  "shipment_duration_days": 3,
  "barcode_checksum_valid": 1,
  "packaging_seal_intact": 1,
  "expiry_date_format_valid": 1,
  "label_language_count": 3,
  "distributor_verified": 1,
  "price_deviation_pct": -2.5,
  "reorder_frequency": 28,
  "lot_number_format_valid": 1,
  "regulatory_approval_code": 4521
}
```
→ `{ "prediction": 0, "label": "GENUINE", "confidence": 1.0, "risk_level": "LOW", "engineered_features": {...} }`

### `POST /api/predict/transaction`
```json
{
  "order_quantity": 480.0,
  "delivery_time_hours": 22.5,
  "invoice_amount_usd": 9800.0,
  "return_rate_pct": 1.8,
  "complaint_count": 1,
  "route_deviation_km": 4.2,
  "customs_clearance_days": 1.9,
  "storage_temp_avg": 5.1
}
```
→ `{ "prediction": 0, "label": "NORMAL", "anomaly_score": 0.0, "risk_level": "LOW", "engineered_features": {...} }`

### Other endpoints
- `POST /api/predict/batch/bulk` — upload a CSV, get predictions for every row
- `POST /api/predict/image` — upload packaging image (requires CNN model)
- `GET /api/health` — server status + which models are loaded
- `GET /api/models/info` — model descriptions, algorithms, performance
- `GET /api/features/batch` / `GET /api/features/transaction` — field definitions for building forms

## Frontend Pages

| Route | Page | Description |
|---|---|---|
| `/` | Overview | Hero, live model status indicators, pipeline explainer |
| `/verify` | Drug Authenticity Verification | Form for all 13 batch fields → Random Forest verdict |
| `/anomaly` | Supply Chain Anomaly Detection | Form for all 8 transaction fields → Isolation Forest verdict |
| `/dashboard` | Results Dashboard | Session stats + filterable history (stored in `localStorage`) |

Both verification pages include one-click "load sample" buttons for genuine/counterfeit and normal/anomaly test data.

## Deployment

- **Backend**: Dockerfile included. Deploy to Render, Railway, or any container host. Set `ALLOWED_ORIGINS` env var to your frontend's domain (comma-separated) — required for CORS with credentials.
- **Frontend**: `npm run build` → deploy `dist/` to Vercel/Netlify. Set `VITE_API_URL` in `.env.production` to your deployed backend URL.
